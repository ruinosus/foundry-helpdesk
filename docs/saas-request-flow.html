Modelo de config04 / 05
Tudo fail-closed

Erros fecham · vazamento é impossível por construção

  • ▰
    Tenant não onboarded / config faltando → erro claro, nunca cai em outro tenant.
  • ▰
    Falha de token / consent → fecha por conexão (o padrão MCP já entregue).
  • ▰
    Vazamento cross-tenant → queries com tenant-scope forçado num único choke point (auditável).
  • ▰
    Domínio sem licença → require_domain() retorna 403 (entitlement, não feature-flag · ADR-010).
O maior risco era o refactor de auth/config — mitigado pela seam SingleTenant primeiro = zero mudança de comportamento, provada pela suíte de evals existente verde antes de ligar o MultiTenant.
