Uma passada barata e barrada para construir.
Uma consulta aparada por identidade para responder.

As duas garantias vivem em momentos diferentes. O gate de fidelidade acontece quando o índice é construído — barato, offline, e barra o que não bate com a fonte. O trim de identidade acontece a cada pergunta no portal — antes do modelo ver qualquer coisa. O elo entre os dois é a base de conhecimento.

🛡️Gate de fidelidade
acontece no build-time
Bundle entra no índice só se ≥80% das citações resolvem para um arquivo que existe. O que não bate vira inspeção, não conhecimento. Nada infiel é servido.
🔒Trim de identidade
acontece no query-time
A poda é por identidade, não por prompt, e ocorre antes do modelo ver qualquer chunk. Jailbreak não amplia acesso. Red-team medido: ASR 0% em 8 ataques.
